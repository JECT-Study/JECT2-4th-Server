# 채팅 기능 도메인 & API 설계 계획

> 기반 디자인: VS 앱 채팅 페이지 (Figma — 채팅 목록 / 채팅방 / 비회원 안내)

---

## 1. 신규 도메인

> **Vote 도메인은 별도 담당자가 설계.** 채팅 도메인에서는 `vote_id` FK만 참조하며,  
> 투표 제목·옵션 텍스트·상태(ONGOING/ENDED)·종료 시간 등은 **Mock** 으로 처리.

### Vote (투표) — ID만 관리

```sql
CREATE TABLE vote (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY
);
```

### VoteParticipation (투표 참여)
채팅 입장 조건. ID 관리만 우선 구현. `selected_option` / `participated_at` 등 상세 필드는 별도 팀원이 추가 예정.

```sql
CREATE TABLE vote_participation (
    id       BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    vote_id  BIGINT NOT NULL REFERENCES vote(id),
    user_id  BIGINT NOT NULL REFERENCES users(id),
    UNIQUE (vote_id, user_id)
);
```

### ChatMessage (채팅 메시지)

```sql
CREATE TABLE chat_message (
    id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    vote_id    BIGINT NOT NULL REFERENCES vote(id),
    sender_id  BIGINT NOT NULL REFERENCES users(id),
    content    TEXT NOT NULL,
    sent_at    TIMESTAMP NOT NULL DEFAULT now()
);

CREATE INDEX idx_chat_message_vote_id ON chat_message (vote_id, sent_at DESC);
```

### ChatRoomUnread (읽음 추적)
안 읽은 메시지 뱃지(최대 99+) 처리에 사용.

```sql
CREATE TABLE chat_room_unread (
    user_id              BIGINT NOT NULL REFERENCES users(id),
    vote_id              BIGINT NOT NULL REFERENCES vote(id),
    last_read_message_id BIGINT,
    last_read_at         TIMESTAMP,
    PRIMARY KEY (user_id, vote_id)
);
```

### User 확장
메시지에 닉네임 + 프로필 아이콘 표시 필요.

```sql
ALTER TABLE users
    ADD COLUMN nickname           VARCHAR(50),
    ADD COLUMN profile_icon_url   VARCHAR(512);
```

## 2. REST API

> 모든 엔드포인트 공통 Header: `Authorization: Bearer {accessToken}`

---

### GET `/api/chats` — 채팅 목록

**Request**
| 구분 | 파라미터 | 타입 | 필수 | 설명 |
|------|----------|------|------|------|
| Query | `status` | `ONGOING \| ENDED` | 필수 | 탭 구분 |

**Response (ONGOING)**
```json
{
  "chats": [
    {
      "voteId": 1,
      "title": "올해 물가가 더 오를까요 안 오를까요",
      "thumbnailUrl": "https://...",
      "optionA": "오른다",
      "optionB": "떨어진다",
      "participantCount": 25,
      "lastMessage": "당연히 오르죠 ㅠㅠ",
      "lastMessageAt": "2026-04-26T14:23:00",
      "endAt": "2026-04-26T18:00:00",
      "unreadCount": 2
    }
  ]
}
```

> `remainingTime`은 내려주지 않음. 프론트에서 `endAt` 기준으로 직접 계산.

**Response (ENDED)**
```json
{
  "chats": [
    {
      "voteId": 2,
      "title": "직장인 점심시간 혼밥 vs 같이먹기",
      "thumbnailUrl": "https://...",
      "optionA": "혼밥이 편하다",
      "optionB": "같이 먹는 게 맞다",
      "participantCount": 23,
      "lastMessage": "팀끼리먹는다거나...",
      "lastMessageAt": "2026-04-20T19:51:00",
      "endAt": "2026-04-20T18:00:00",
      "unreadCount": 0
    }
  ]
}
```

정렬: ONGOING → `lastMessageAt DESC` / ENDED → `endAt DESC`

|------|----------|------|------|------|

### GET `/api/chats/{voteId}/messages` — 메시지 목록

**Request**
| 구분 | 파라미터 | 타입 | 필수 | 설명 |
|------|----------|------|------|------|
| Path | `voteId` | `Long` | 필수 | 투표 ID |
| Query | `cursor` | `Long` | 선택 | 이전 페이지 마지막 messageId. 없으면 최신부터 |
| Query | `size` | `Int` | 선택 | 페이지 크기 (기본값: 30) |

**Response**
```json
{
  "messages": [
    {
      "messageId": 1234,
      "content": "당연히 오르죠",
      "sentAt": "2026-04-26T14:23:00",
      "senderNickname": "슈퍼강아지_485",
      "senderProfileIcon": "https://cdn.example.com/icons/dog.png",
      "senderVoteOption": "A",
      "isMine": false
    }
  ],
  "nextCursor": 1200,
  "hasNext": true
}
```

### POST `/api/chats/{voteId}/read` — 읽음 처리

**Request**
| 구분 | 파라미터 | 타입 | 필수 | 설명 |
|------|----------|------|------|------|

## 3. WebSocket (STOMP)

```
연결 엔드포인트: ws://.../ws

─ 채팅 메시지 ──────────────────────────────────────
구독: /topic/chat/{voteId}         ← 새 메시지 수신 (모든 참여자)

─ 읽지 않은 메시지 뱃지 ────────────────────────────
구독: /topic/chat/{voteId}/unread  ← 새 메시지 도착 시 unreadCount 갱신
                                      (채팅 목록 화면에서 실시간 반영)
```

**수신 Payload (`/topic/chat/{voteId}`):**
```json
{
  "messageId": 1234,
  "content": "당연히 오르죠",
  "sentAt": "2026-04-26T14:23:00",
  "senderNickname": "슈퍼강아지_485",
  "senderProfileIcon": "https://cdn.example.com/icons/dog.png",
  "senderVoteOption": "A",
  "isMine": false
}
```

**수신 Payload (`/topic/chat/{voteId}/unread`):**
```json
{ "unreadCount": 3 }
```

> **REST vs WebSocket 역할 분리**
> - 메시지 전송 → REST `POST /api/chats/{voteId}/messages`
> - 최초 진입 시 과거 메시지 조회 → REST `GET /api/chats/{voteId}/messages`
> - 실시간 메시지 수신 → WebSocket 구독 전용

## 4. 권한 정책

| 상황 | 처리 |
|------|------|
| 비회원 → 채팅 탭 접근 | `401` → 프론트 비회원 안내 바텀시트 |
| 투표 미참여 회원 → 메시지 전송 | `403` |
| 투표 종료 상태 → 메시지 전송 | `403` (채팅 마감) |
| 투표 미참여 회원 → 채팅 조회 | 허용 (읽기 전용) |

## 5. 구현 우선순위

| 단계 | 내용 |
|------|------|
| 1단계 | VoteParticipation + User(nickname, profileIcon) 확장 (Vote는 mock 처리) |
| 2단계 | 채팅 목록 REST API (`GET /api/chats`) |
| 3단계 | 채팅방 REST API (메시지 CRUD + 읽음 처리) |
| 4단계 | WebSocket/STOMP 설정 + 실시간 메시지 수신/전송 |
| 5단계 | 게이지바 폴링 API + unreadCount 실시간 갱신 (WebSocket) |
